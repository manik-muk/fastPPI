How fastPPI works

1. Trace your Python preprocessing

fastPPI runs your existing preprocessing code under a tracer that observes your pandas and string operations, building a high-level graph of your pipeline.

2. Generate optimized native code

The traced graph is compiled into a low-level representation and emitted as a native library / binary, leveraging efficient C implementations of core operations.

3. Run as a fast, self-contained binary

The resulting artifact can be called from Python, invoked from the CLI, or embedded into existing services to reduce end-to-end latency and stabilize performance.
